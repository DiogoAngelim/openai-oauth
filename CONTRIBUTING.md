# Contributing Guide

Thank you for considering contributing to this project! To keep the codebase clean, secure, and maintainable, please follow these guidelines:

## Getting Started

- Fork the repository and create your branch from `main`.
- Use descriptive branch names (see BRANCHING.md).
- Ensure your branch is up to date with `main` before submitting a PR.

## Code Quality

- Write clear, maintainable code and include comments where helpful.
- Run linting and tests locally before pushing.
- Follow the existing code style and conventions.

## Pull Requests

- Submit a pull request (PR) for review.
- Link related issues in your PR description.
- Ensure all CI/CD checks pass before requesting review.
- Address review comments promptly.

## Security

- Never commit secrets or sensitive data.
- Report security issues privately (see SECURITY.md).

## Branching

- Keep feature and fix branches short-lived.
- Delete branches after merging.

## Issues

- Search for existing issues before opening a new one.
- Provide clear, detailed information when reporting bugs or requesting features.

---

_Your contributions help make this project better! Thank you for following these guidelines._
